## 审核规则

下列表述视为「过度业务承诺」，命中即应记录为 issue：

### 1. 保本类承诺（severity: high）
- 「保本」「不亏损」「本金无忧」「绝对安全」「保证本金」「兜底」

### 2. 收益类承诺（severity: high）
- 「保收益」「保证收益」「最低收益 X%」「锁定收益」「稳赚」「稳赚不赔」「躺赚」「包赚」

### 3. 确定性未来预测（severity: medium ~ high）
- 「一定上涨」「必涨」「肯定盈利」「将会大涨」「未来 N 个月必定上涨 X%」「确定性机会」「板上钉钉」

### 4. 业绩排名 / 比较类承诺（severity: medium）
- 「行业第一」「排名最高」「收益率最高」「跑赢所有同类」「业内最优」
- 缺少时间区间或基准的收益率宣传（如「年化 30%」未注明区间）

### 5. 推荐买入类（severity: medium）
- 「建议购买」「推荐买入」「赶紧上车」「不买就亏了」「现在是最佳时机」

### 6. 其它诱导性表述（severity: low ~ medium）
- 过于绝对的修饰语：「绝对」「百分百」「毫无风险」
- 隐瞒风险的措辞：「闭眼买」「无脑买」

## 🅱️ 错别字 / 文字错误 类别（severity: low，默认）

**仅当你非常确定某个字写错时才报告。** 重点关注：

### 7.1 同音字误用（参考型 - 仅供你识别"该字写错"这一类模式，不是要在用户文本里找的目标字）
模式：拼音相同但字形不同的字被误用。识别原则是**理解上下文之后判断「这里该用 A 但作者写了 B」**。

### 7.2 形近字误用（参考型）
模式：字形高度相似但意义不同的字被误用，常见于手写 OCR 后或快速输入时。

### 7.3 多字 / 少字 / 字符缺失
- 明显多了或少了字，导致语句不通顺（必须 `numbered_text` 里**实际**有这种字句才报告）
- **年份数字缺位**：形如 "<少于 4 位数字>年" 的非标准年份写法（标准年份应为 4 位数字 + 年，如 "2024 年" "2025 年"）。**仅当 numbered_text 中实际出现位数异常的年份时才报告** → **severity = `medium`**
- **无意义重复字符序列**：连续 ≥ 3 个相同字母（或同一字符）拼接成的无语义片段，明显属于占位符未替换或录入残留。**仅当 numbered_text 中实际出现此类序列时才报告**：
  - 若该序列**嵌在有意义内容中间，替换掉了原本应有的词**→ **severity = `medium`**
  - 若该序列**仅出现在句末作为悬尾噪声**→ **severity = `low`**

### 7.4 数字与单位错误
- 半角全角混用（如百分号风格混乱）
- 数字内部不必要的空格（如把 "1.5%" 写成 "1 .5%"）
- 中英文标点混用（中文句子末尾误用英文句号）
- **年份格式异常 / 缺少世纪位数字**：与 §7.3 规则同步，正确年份应为完整 4 位数字 → **severity = `medium`**

### 7.5 明显标点错误
- 双句号 `。。`、双感叹号在严肃语境
- 句中用了英文逗号 `,` 而非中文 `，`

### ⛔ 严禁误报的情况（白名单）

- **行业术语 / 专业缩写**一律不报：权益类、久期、Beta 系数、Alpha、α、β、PE、PB、PEG、ROE、CAGR、Q1/Q2、AH 股、北向资金、险资、LP/GP …
  - ⚠️ **白名单仅保护已知的专业术语缩写**。**连续 ≥ 3 个相同字母组成的无意义序列**不属于专业术语，不受白名单保护；但必须**先确认 numbered_text 里真有这种序列**，再按 §7.3 规则照常报告。
  - 判断准则：能在主流金融/经济词典里查到的视为术语；纯字母无规律重复的一律不视为术语。
- **人名 / 基金名 / 产品名 / 公司名 / 指数名**不报
- **数据风格差异**不报：`0.3%` 与 `0.30%` 都对、`30%` 与 `百分之三十` 都对
- **古汉语 / 书面化用法**不报：「之于」「于此」「殆」「乃」「兹」
- **拿不准的字一律不报**，**宁可漏报不可错报** ⚠️ **此原则仅适用本节 §7 错别字判断；合规相关的 §1–§6 必须严格按规则报告，不允许借口"拿不准"漏报**

### 错别字 issue 输出格式

- `category`：固定填 `"错别字"`
- `severity` 校准（请严格遵守）：
  - 默认 `low`
  - **年份数字缺位**（少于 4 位的"年份+年"写法）→ `medium`
  - **重复字符乱码嵌在有意义内容中**（替换了原本应有的词）→ `medium`
  - **重复字符乱码仅作句末悬尾噪声** → `low`
  - 其它导致严重歧义的错字 → `medium`
- `excerpt`：填该句**原文**
- `comment`：必须给出 `"X 应改为 Y"` 形式的明确修订建议（X 是 numbered_text 里出现的原始片段，Y 是建议改写后的正确版本）
